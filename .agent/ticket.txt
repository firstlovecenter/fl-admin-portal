ContextDependabot PR #540 bumps recharts 2.13.3 → 3.9.2 in web-react-ts. It was HELD during the 2026-07-08 Dependabot safe-merge sweep because it is a genuine breaking migration whose failure modes are runtime-behavioral and invisible to the current CI gates.Two classes of breakage were found when applying recharts 3:Bar.onClick payload shape changed — the dangerous one (drill-down break). In recharts 3, Bar.onClick is (data: BarRectangleItem, index, event) => void and the original datum now lives under data.payload. In web-react-ts/src/components/ChurchGraph/ChurchGraph.tsx, handleBarClick(data: any, statKey) reads data.id, data.category, data.week, data.year directly (and spreads ...data into clickCard). Because the param is typed any, tsc gives zero safety — the build stays green while the dashboard drill-down silently breaks (clicking a bar would read undefined fields). web-react-ts/src/pages/dashboards/TrendSpark.tsx has the same pattern in its two Bar.onClick handlers (there they are typed, so tsc flags them).LabelList.content typing changed. recharts 3's LabelContentType = ReactElement | ((props: Props) => RenderableText | ReactElement). The custom renderBarLabel functions in both ChurchGraph.tsx and TrendSpark.tsx (typed with the local ChartBarLabelProps and returning JSX.Element | null) no longer type-check (6 TS errors total across the two files).Critically, there is no test coverage for the chart components — grep confirms no test file references ChurchGraph, TrendSpark, or recharts. So neither vite build nor the vitest suite would catch the broken drill-down; only a manual/visual click of the dashboards would. This is why the bump cannot be shipped under a build-only gate.Sibling: SYN-200 (Apollo 4 / graphql 17).
